opencv 图片基本操作

发表于 2017-07-23 | 分类于 tool | | 阅读次数

对图片像素的操作实际是对图片读入后返回的numpy数组进行操作，记住数组格式是[height, width, channels]其中channels是BGR顺序排列。这些对像素的操作归到了对numpy数组的操作。考虑到速度可以使用numpy数组的img.item()或者img.itemset()方法来设置。在numpy中对数组的img.shape，img.size，img.dtype 操作都是可用的，且很有用。

图片加法

图片数组(uint8)之间的加法，最好使用cv2.add(x,y)因为图片数组一般的大小在0到255之间，如果加的结果操作了255那么cv2.add会取255，如果是一般的numpy数组加法得到的就是4了。
图片加法还存在融合的形式，两个图片按照比例叠加在一起cv2.addWeighted(img1, 0.7, img2, 0.3, 0)最后是常数项$dst=\alpha \cdot img1 + \beta \cdot img2 + \gamma$。

程序处理效率

使用cv2.getTickCount()来获取开始和结束点，然后除频率可以得到程序执行时间。

图片尺寸改变

使用res = cv2.resize(img,None,fx=2, fy=2, interpolation = cv2.INTER_CUBIC)或者res = cv2.resize(img,(2*width, 2*height), interpolation = cv2.INTER_CUBIC)都是可以将图片放大为原有尺寸的两倍。

特征提取和描述

对于一个图片它独一无二的地方一般是它的边边角角，像是以其连成一块的色彩往往不能表示这个图片。接下来我们要做的就是提取图片的特征并给予它恰当的描述。这样根据特征和描述就可以对两张图片进行匹配等操作。

Harris Corner Detection

在一个图片上将大小固定的窗口在各个方向上移动如果窗口灰度出现大的变化，则这个很可能就是角点。
公式：$E(u,v)=\sum _(x,y) w(x,y)(I(x+u,y+v)-I(x,y))^2$
在opencv中cv2.cornerHarris()实现了这个功能，参数有。

  • img-输入图片，需要是float32类型的灰度图
  • blockSize-角点领区大小
  • ksize-Sobel的孔径参数(aperture parameter)，也就是Sobel核的半径，如1、3、5、7，用来求导的。
  • k-k是一个经验参数用来调节阀值，一般取值在0.04-0.06之间。

优点：具有旋转不变性。
缺点：不具有尺度不变性。

参考 Harris角点
参考 Harris角点检测(Python-OpenCV)

注意：在显示一个图片定位的角点的时候一般会设置通道颜色。对于matplotlib是RGB的顺序，对于cv2是BGR的顺序。所以可能出现颜色不一致的情况。

Shi-Tomasi Corner Detector

Shi-Tomasi Corner Detector是对Harris的优化，将以前的$R=\lambda _1\lambda _2 - k(\lambda _1+\lambda _2)^2$修改为$R=min(\lambda _1, \lambda _2)$。优化后可以提供更好的结果。
对应函数是cv2.goodFeaturesToTrack()它的参数是一张灰度图片，指定抽取的角点数，最小角点阀值(大于这个数字才进入候选)，角点最小距离(某个角点最小距离范围内如果有更强的角点则删除本角点)

SIFT

SIFT全称Scale-Invariant Feature Transform，在上面提到的两个算法中它们都具有旋转不变性，但是对于大小发生改变的图像就不能识别同一个角点了。下面我们要学习的算法将具有scale invariant(尺度不变性)。这个是专利保护需要收费的模块。
sift = cv2.xfeatures2d.SIFT_create():创建SIFT特征提取器。
kp = sift.detect(gray,None):进行关键点提取。
img=cv2.drawKeypoints(gray,kp,img):绘制关键点。

算法解释参考:SIFT算法详解

SURF

相比SIFT，可以更快的提取关键点。
surf = cv2.xfeatures2d.SURF_create(400):创建surf，包含参数是设置Hessian阀值。
kp, des = surf.detectAndCompute(img,None):计算关键点

FAST

这是更快的算法，可以满足实时的要求。在opencv3中部分接口发生了改变。
fast = cv2.FastFeatureDetector()改变为fast = cv2.FastFeatureDetector_create().
fast.setBool('nonmaxSuppression',0)改变为fast.setNonmaxSuppression(0).
img3 = cv2.drawKeypoints(img, kp, color=(255,0,0))改变为img3 = cv2.drawKeypoints(img, kp, None, color=(255,0,0)).
在opencv中很多API接口迁移走了，需要按照contrib才能使用。重新安装后记得重新复制cv2.cpython-35m-darwin.so文件。

BRIEF

更快的速度，更少的内存占用，更多的关键点。
star = cv2.xfeatures2d.StarDetector_create():初始化探测器。
brief = cv2.xfeatures2d.BriefDescriptorExtractor_create():创建提取器。
kp = star.detect(img,None):执行探测。
kp, des = brief.compute(img, kp):计算关键点。

ORB

在opencv3中ORB模块函数由cv2.ORB()改为orb = cv2.ORB_create()。
kp = orb.detect(img,None):进行探测。
kp, des = orb.compute(img, kp):计算关键点。

特征匹配

查找两张图片中特征匹配的点。当然这里需要读入两张图片，同时创建关键点探测器，并计算得到关键点。然后创建Matcher来比较关键点，最后绘制出来。
kp1, des1 = orb.detectAndCompute(img1,None)返回的参数包括来关键点和描述。
bf = cv2.BFMatcher(cv2.NORM_HAMMING, crossCheck=True)创建匹配器。第一个参数是计算距离的方式。
img3 = cv2.drawMatches(img1,kp1,img2,kp2,matches[:10], flags=2)生成两个图片特征匹配的图片。
